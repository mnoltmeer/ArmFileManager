***АС «Менеджер файлів АРМ ВЗ» 1.0.1.21***

Добавлено:

- функция интерпретатора _WriteConnCfg(pID, pPrm, pVal) - позволяет вносить изменения в файл конфига подключения, основываясь на его ИД. Была добавлена из-за того, что парсер интерпретатора не совсем корректно обрабатывает пути к файлам, которые содержат знак "_".

- новые параметры RegExDL и RegExUL в конфиг подключений, позволяют использовать в шаблонах, указанных в параметрах DownloadFileMask и UploadFileMask регулярные выражения.

Пояснение: дело в том, что при отправке файлов Программа фильтрует нужные файлы по шаблонам. Но Windows (или, во всяком случае, функция FindFirst() из состава WinAPI) считает шаблон ??? (три одиночных символа) эквивалентным шаблону * (любое количество символов). Из-за этого, при выгрузке файлов для Товарооборота, на сервер выгружались лишние файлы (конкретно файлы мониторинга). Использование регулярных выражений позволит избежать этого. О регулярных выражениях можно почитать в интернете. В той же Википедии есть достаточно информативная статья.

Важно: при использовании регулярных выражений шаблон типа *.dbf не работает (вернее работает, но по-другому), так что или вы выбираете просто файлы по маске или используете правильный синтаксис регулярных выражений.

***АС «Менеджер файлів АРМ ВЗ» 1.0.0.7***

Исправлено:

- при завершении работы Программы, закрытие подключений выполнялось дважды, что приводило к появлению в логе сообщений типа: "Помилковий ID потоку , поток: 0".

- даже при отсутствии файлов для загрузки выводилось сообщение о том, что были загружены новые файлы.

- при перечитывании конфига Программы в конфигах подключений не обновлялись значения параметров, отвечающих за настройки сервера статистики.

- пустые значения параметров BackUpDirDl и BackUpDirUl приводили к ошибке создания директории при инициализации подключения даже если BackUpDl и BackUpUl были установлены в 0.

- записи об обработанных файлах выводились в неудобочитаемом формате типа "09:00:00 1.txt". Теперь формат вывода такой: "09:00:00| 1.txt"


Добавлено:

- новый параметр StationID в конфиг Программы, идентификатор рабочего места.

- теперь параметр IndexVZ в конфиге Программы содержит индекс ВПЗ.
 
- в параметрах конфиг-файлов подключений можно использовать следующие переменные:
	$IndexVZ - индекс ВПЗ 
	$StationID - идентификатор рабочего места
	$RegionID - идентификатор региона
	$Date - текущая дата в формате dd.mm.yyyy
	$AppPath - полный путь к каталогу, из которого запущена Программа.
Вместо этих переменных будут подставлены значения соответствующих параметров из конфига Программы. Например, если указать в конфиге подключения строку типа "LocDirDl=$AppPath\$IndexVZ" это приведет к тому, что загружаемые файлы будут сохраняться в каталоге с именем соответствующим индексу ВПЗ и расположенном в рабочем каталоге Программы.

- новый параметр RunOnce в конфиг подключений, если его значение установлено в 1, подключение выполнит один сеанс обмена, после чего остановится. При этом его все еще можно запустить вручную.

- новый параметр StartAtTime в конфиг подключений, если его значение установлено в 1, подключение выполнит сеанс обмена после того, как наступит время, указанное в параметре TimeStart.

- новый параметр TimeStart в конфиг подключений. Содержит время в формате hh:mm, при наступлении которого будет выполнен сеанс обмена. Если на момент запуска подключения системное время уже больше, чем в параметре TimeStart, сеанс обмена будет выполнен немедленно.
